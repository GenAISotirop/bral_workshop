Πριν ξεκινήσουμε

Αυτό το φύλλο εργασίας σας καθοδηγεί βήμα–βήμα μέσα από έναν πλήρη κύκλο BRaL. Θα δημιουργήσετε μια διαδραστική προσομοίωση του νόμου του Hooke, ακριβώς όπως θα την κάνατε με τους μαθητές σας. Σε κάθε βήμα θα βρείτε έτοιμα prompts για copy/paste, αναμενόμενες απαντήσεις και σημεία παρέμβασης ως εκπαιδευτικός.

✅ Τι χρειάζεστε ανοιχτό (σε 3 tabs)

  • Tab 1: Αυτό το φύλλο εργασίας (για να το ακολουθείτε)
  • Tab 2: claude.ai — Logged in (για Build & Learn)
  • Tab 3: chatgpt.com — Logged in (για Refine)

Γιατί δύο εργαλεία;

Το Claude έχει Artifacts — ο κώδικας τρέχει μέσα στο chat, χωρίς copy/paste σε external editor. Ιδανικό για Build (δημιουργία) και Learn (αναστοχασμός).

Το ChatGPT έχει πιο γενναιόδωρα όρια στη δωρεάν έκδοση και δίνει "fresh perspective" για το Refine (debugging). Έτσι εξοικονομούμε τα Claude credits για τη δημιουργική δουλειά.

💡 Αυτή η λογική είναι αντιγράψιμη: οι μαθητές σας θα μάθουν ότι διαφορετικά εργαλεία ΤΝ εξυπηρετούν διαφορετικούς σκοπούς — σημαντική δεξιότητα AI literacy.

01
⚙ SETUP

Ορίζω το φαινόμενο & τον στόχο

Defining phenomenon & learning goal
⏱ 5'

📝 Οδηγίες

Πριν αγγίξετε οποιοδήποτε AI, συμπληρώστε αυτά τα στοιχεία. Μην παραλείψετε αυτό το βήμα — είναι η καρδιά του BRaL.

  1. Φαινόμενο: Νόμος του Hooke (ταλάντωση ελατηρίου)
  2. Τάξη: Α' Λυκείου ή Β' Γυμνασίου (ανάλογα)
  3. Μαθησιακός στόχος: Οι μαθητές να ανακαλύψουν τη γραμμική σχέση F = -kx και την περίοδο ταλάντωσης T = 2π√(m/k)
  4. Στοχευμένη παρανόηση: «Όσο βαρύτερη η μάζα, τόσο πιο γρήγορη η ταλάντωση» — στην πραγματικότητα βραδύτερη!

Σημείο παρέμβασης εκπαιδευτικού

Στην τάξη σας, μην δώσετε έτοιμα τα παραπάνω στους μαθητές. Κάντε τους να τα βρουν μόνοι τους με ερωτήσεις:

  • «Τι θέλετε να ανακαλύψουν οι μαθητές σας;»
  • «Ποια παρανόηση έχετε παρατηρήσει συχνά;»
  • «Ποιες μεταβλητές είναι σημαντικές;»
Πιθανή δυσκολία μαθητών: Θα πουν «θέλω να μάθουν για τα ελατήρια». Πιέστε για ΠΙΟ ΣΥΓΚΕΚΡΙΜΕΝΟ στόχο — τι ακριβώς να ανακαλύψουν.
Σύνδεση με BRaL Στο BRaL η επιστημονική σκέψη προηγείται της ΤΝ. Οι μαθητές είναι "model builders" — πρέπει να ξέρουν τι μοντελοποιούν και γιατί, πριν ζητήσουν από την ΤΝ να γράψει κώδικα (Sotiropoulos, 2026).
02
◆ BUILD

Κάνω πρόβλεψη πριν τρέξω

Prediction before execution
⏱ 5'

📝 Οδηγίες

Πριν δημιουργήσετε οτιδήποτε, γράψτε τι περιμένετε να συμβεί. Αυτό δεν είναι διακοσμητικό βήμα — είναι η βάση για μετέπειτα έλεγχο.

Πρόβλεψη 1: Αν αυξήσω τη μάζα (m), η περίοδος T θα...

Πρόβλεψη 2: Αν αυξήσω τη σταθερά ελατηρίου (k), η περίοδος T θα...

Πρόβλεψη 3: Αν διπλασιάσω την αρχική μετατόπιση (x₀), η περίοδος T θα...

✓ Τι περιμένουμε (σωστές απαντήσεις)

Πρόβλεψη 1: Θα αυξηθεί (T ∝ √m). Πολλοί μαθητές θα πουν «θα μειωθεί» — κλασική παρανόηση!

Πρόβλεψη 2: Θα μειωθεί (T ∝ 1/√k). Σκληρότερο ελατήριο = γρηγορότερη ταλάντωση.

Πρόβλεψη 3: ΔΕΝ θα αλλάξει! (Η περίοδος είναι ανεξάρτητη από το πλάτος στην απλή αρμονική ταλάντωση). Αυτό είναι counterintuitive — πολλοί θα το μαντέψουν λάθος.

Σημείο παρέμβασης εκπαιδευτικού

ΜΗΝ διορθώσετε τώρα! Αφήστε τους να δουν τι λένε — ακόμα και αν είναι λάθος. Οι λάθος προβλέψεις θα γίνουν καύσιμο μάθησης αργότερα όταν τρέξουμε την προσομοίωση.

💡 Tip: Ζητήστε από κάθε ομάδα να συμφωνήσει πριν γράψει — αν υπάρχει διαφωνία, αυτή η συζήτηση είναι χρυσάφι.
Σύνδεση με BRaL "Predictive reasoning" πριν την εκτέλεση — αυτό μετατρέπει το "τρέχω και βλέπω" σε συνειδητή μάθηση αντί τυχαίο παιχνίδι. Είναι ρητά απαιτούμενο στο BRaL framework.

◆ BUILD

Στέλνω το πρώτο prompt στο Claude

Initial prompt to Claude
⏱ 10'

📝 Οδηγίες

  1. Πηγαίνετε στο Claude.ai (Tab 2)
  2. Ξεκινήστε νέα συνομιλία
  3. Αντιγράψτε το παρακάτω prompt και επικολλήστε το στο Claude
  4. Πατήστε Enter και περιμένετε
📋 PROMPT #1 · BUILD
Δημιούργησε ένα HTML artifact με διαδραστική προσομοίωση για τον νόμο του Hooke (ταλάντωση ελατηρίου-μάζας). ΠΡΟΔΙΑΓΡΑΦΕΣ: - Κάθετο ελατήριο με μάζα που κρέμεται - 3 sliders: * Σταθερά ελατηρίου k (10-200 N/m) * Μάζα m (0.1-2 kg) * Αρχική μετατόπιση x₀ (-0.5 έως 0.5 m) - Κουμπί Play/Pause και Reset - Εμφάνιση σε πραγματικό χρόνο: * Θέση x(t) * Ταχύτητα v(t) * Περίοδος T * Κινητική + Δυναμική ενέργεια (μπάρες) - Γραφική παράσταση x(t) σε πραγματικό χρόνο - Εξίσωση F = -kx ορατή στην οθόνη ΤΕΧΝΙΚΑ: - Single HTML file, χωρίς εξωτερικές βιβλιοθήκες - Χρώματα: κόκκινο ελατήριο, μπλε μάζα - Labels στα Ελληνικά - Σχόλια στον κώδικα στα Ελληνικά ΕΚΠΑΙΔΕΥΤΙΚΟ ΠΛΑΙΣΙΟ: - Για μαθητές Α' Λυκείου - Στόχος: ανακάλυψη της σχέσης T = 2π√(m/k) - Παρανόηση που στοχεύω: «βαρύτερη μάζα = γρηγορότερη ταλάντωση»

✓ Τι αναμένουμε από το Claude

  • Θα δημιουργήσει ένα Artifact (δεξιά πλευρά της οθόνης)
  • Θα βλέπετε την προσομοίωση να τρέχει άμεσα μέσα στο Claude
  • Θα δώσει σύντομη εξήγηση του κώδικα
  • Χρόνος απόκρισης: 30-60 δευτερόλεπτα

⚠ Πιθανές δυσκολίες

  • «Δεν βλέπω Artifact»: Κοιτάξτε δεξιά — μπορεί να είναι minimized. Πατήστε πάνω στο preview.
  • Το ελατήριο δείχνει "περίεργο": Είναι ΟΚ! Θα το διορθώσουμε στο Refine — είναι μέρος της διαδικασίας.
  • Δεν τρέχει η ταλάντωση: Δοκιμάστε να σύρετε τη μάζα και αφήστε την. Αν δεν δουλεύει, το εντοπίζουμε στο Refine.
  • «Ο κώδικας είναι πολύ μεγάλος»: Δεν πειράζει — δεν θα τον διαβάσετε. Εστιάζουμε στη συμπεριφορά.

Σημείο παρέμβασης εκπαιδευτικού

Στην τάξη σας: Αφήστε τους μαθητές να γράψουν το δικό τους prompt πρώτα (με καθοδήγηση). Το παραπάνω prompt είναι πολύ λεπτομερές — ξεκινήστε με απλούστερη έκδοση για τους μαθητές.

Απλούστερο prompt για μαθητές:

«Φτιάξε προσομοίωση ελατηρίου με μάζα. Θέλω sliders για τη μάζα και τη σταθερά. Δείξε μου την ταλάντωση και την περίοδο.»

Πιθανή δυσκολία: Οι μαθητές θα ξεχάσουν σημαντικά στοιχεία (units, εκπαιδευτικό πλαίσιο). Αυτό είναι διδακτική ευκαιρία — δείξτε τους ότι το AI "ξεχνά" αν δεν ζητήσετε συγκεκριμένα.
Σύνδεση με BRaL Το Claude εδώ λειτουργεί ως "on-demand coder" — μεταφέρει το γνωστικό φορτίο από τη σύνταξη κώδικα στη μοντελοποίηση. Οι μαθητές εστιάζουν στη φυσική, όχι στη σύνταξη JavaScript.

📋 Checklist ελέγχου

✓ Τι πιθανότατα θα παρατηρήσετε

  • ✅ Οι προβλέψεις επιβεβαιώνονται — εκεί βασίζεται η μάθηση των μαθητών!

Σημείο παρέμβασης εκπαιδευτικού

  1. «Είδατε ποτέ στην πραγματικότητα ελατήριο που ταλαντώνεται για πάντα;»
  2. «Γιατί πιστεύετε ότι σταματά στην πραγματικότητα;»
  3. «Τι λείπει από το μοντέλο μας;»
Αναμενόμενες απαντήσεις μαθητών: «τριβή», «αντίσταση του αέρα», «χάνει ενέργεια». Όλες οδηγούν στην ίδια έννοια: damping.
Σύνδεση με BRaL Αυτό το βήμα είναι το σημείο-καμπή μεταξύ Build και Refine. Οι μαθητές δεν είναι "passengers" σε έτοιμη προσομοίωση — είναι "drivers" που αναγνωρίζουν ότι το μοντέλο τους έχει κενά και θέλει βελτίωση.

◆ REFINE

Μεταφέρω τον κώδικα & ζητάω sanity checks

Transfer code & request sanity checks
⏱ 10'

📝 Οδηγίες

  1. Επιστροφή στο Claude (Tab 2)
  2. Στο Artifact, πατήστε το κουμπί Copy (πάνω δεξιά στο preview)
  3. Πηγαίνετε στο ChatGPT (Tab 3) — νέα συνομιλία
  4. Επικολλήστε το παρακάτω prompt ΚΑΙ τον κώδικα μαζί
📋 PROMPT #2 · REFINE / SANITY CHECK
Έχω έναν HTML/JavaScript κώδικα εκπαιδευτικής προσομοίωσης για τον νόμο του Hooke (ελατήριο-μάζα) που δημιούργησε άλλο AI tool. ΑΚΟΛΟΥΘΩ ΤΗ ΜΕΘΟΔΟΛΟΓΙΑ BRaL (Build-Refine-Learn) και είμαστε τώρα στο REFINE phase. ΧΡΕΙΑΖΟΜΑΙ: 1. Να εκτελέσεις 3 "sanity checks" για φυσική ορθότητα: α) Στο x=0 είναι F=0; β) Όταν διπλασιάζω το k, η δύναμη για ίδιο x διπλασιάζεται; γ) Η συνολική ενέργεια διατηρείται σταθερή (χωρίς damping); 2. Να εντοπίσεις συγκεκριμένα προβλήματα: - Perpetual motion (η ταλάντωση δεν σταματά ποτέ) - Πιθανή "ενεργειακή ολίσθηση" (energy drift) - Άλλα προβλήματα που βλέπεις 3. Να δώσεις ΜΟΝΟ patches (τι γραμμές να αλλάξω), ΟΧΙ ξαναγράψιμο όλου του κώδικα. 4. Να συνδέσεις κάθε διόρθωση με τον φυσικό νόμο/αρχή. ΣΗΜΑΝΤΙΚΟ: Μη λύσεις όλα τα προβλήματα μονομιάς. Εστίασε πρώτα στα 3 sanity checks και τον εντοπισμό. ΚΩΔΙΚΑΣ: [ΕΠΙΚΟΛΛΗΣΕ ΕΔΩ ΟΛΟ ΤΟΝ ΚΩΔΙΚΑ ΑΠΟ ΤΟ CLAUDE]

✓ Τι αναμένουμε από το ChatGPT

Το ChatGPT συνήθως θα:

  • Εκτελέσει νοητά τα 3 sanity checks και θα σας πει τι βρήκε
  • Εντοπίσει το perpetual motion ως κύριο πρόβλημα
  • Επισημάνει πιθανή energy drift λόγω numerical integration (συνήθως Euler method)
  • Προτείνει πιο ακριβή μέθοδο (π.χ. Verlet integration) ή προσθήκη damping
  • Δώσει συγκεκριμένες γραμμές κώδικα που πρέπει να αλλάξουν

⚠ Πιθανές δυσκολίες

  • «Το ChatGPT μου δίνει ολόκληρο τον κώδικα ξανά»: Φυσιολογικό — το ChatGPT τείνει να επαναγράφει. Αγνοήστε το full code και εστιάστε στα patches.
  • «Δεν βρίσκει πρόβλημα»: Πιθανόν το prompt δεν πέρασε σωστά. Βεβαιωθείτε ότι επικολλήσατε και τον κώδικα.
  • «Ο κώδικας είναι πολύ μεγάλος, δεν χωράει»: Το ChatGPT free έχει context limit. Επικολλήστε μόνο το κύριο JavaScript block.

Σημείο παρέμβασης εκπαιδευτικού

Στους μαθητές, μην εξηγήσετε τι είναι "Euler method" ή "Verlet integration". Αρκεί να καταλάβουν: «ο υπολογιστής έχει ένα μικρό λάθος κάθε φορά που υπολογίζει — και αυτά μαζεύονται».

Εστιάστε στο physics insight: «γιατί τα πραγματικά ελατήρια σταματούν;»

💡 Tip: Αν το ChatGPT δώσει πολύ τεχνική απάντηση, ζητήστε απλοποίηση: "Εξήγησέ το ώστε να το καταλάβει μαθητής Α' Λυκείου."
Σύνδεση με BRaL Refine ≠ debugging. Είναι validation (έλεγχος συμφωνίας με νόμους φυσικής), diagnosis (γιατί αποκλίνει) και justification (σύνδεση με θεωρία). Χωρίς αυτό, το Refine γίνεται τυφλό trial-and-error.
06
◆ REFINE

Προσθέτω damping (επέκταση μοντέλου)

Model extension — adding damping
⏱ 10'

📝 Οδηγίες

  1. Παραμείνετε στο ChatGPT (ίδια συνομιλία)
  2. Επικολλήστε το παρακάτω prompt
  3. Αντιγράψτε τον νέο κώδικα ή τα patches που θα δώσει
📋 PROMPT #3 · MODEL EXTENSION
Τέλεια! Τώρα ας προσθέσουμε damping (απόσβεση) στο μοντέλο μας, ώστε να είναι πιο ρεαλιστικό. ΖΗΤΑΩ: 1. Πρόσθεσε έναν τέταρτο slider για damping coefficient (c): - Εύρος: 0 έως 2 N·s/m - Default value: 0.3 2. Τροποποίησε την εξίσωση κίνησης: Από: F = -kx Σε: F = -kx - c·v (c·v είναι η δύναμη απόσβεσης) 3. Πρόσθεσε ένα label που δείχνει σε ποιο regime είμαστε: - Underdamped (c² < 4mk): "Υποκρίσιμη απόσβεση" - Critically damped (c² = 4mk): "Κρίσιμη απόσβεση" - Overdamped (c² > 4mk): "Υπερκρίσιμη απόσβεση" 4. Δώσε μου τον ΠΛΗΡΗ νέο κώδικα (τώρα χρειάζομαι ολόκληρο) 5. Σε 2-3 γραμμές: εξήγησε ποια φυσική αρχή ενσωματώσαμε.

🔄 Επιστροφή στο Claude

  1. Αντιγράψτε τον νέο κώδικα από το ChatGPT
  2. Πηγαίνετε πίσω στο Claude (Tab 2)
  3. Στην ίδια συνομιλία, στείλτε:
📋 PROMPT #4 · UPDATE CLAUDE ARTIFACT
Ενημέρωσε το artifact μας με τον παρακάτω κώδικα (έχει ενσωματωμένη απόσβεση/damping). Διατήρησε τη δομή του artifact: [ΕΠΙΚΟΛΛΗΣΕ ΕΔΩ ΤΟΝ ΝΕΟ ΚΩΔΙΚΑ ΑΠΟ CHATGPT]

✓ Τι να δοκιμάσετε τώρα

Ρυθμίστε: k=100 N/m, m=1 kg και πειραματιστείτε με το damping c:

  • c = 0 → Συνεχής ταλάντωση (όπως πριν)
  • c = 0.5 → Υποκρίσιμη — ταλάντωση που "σβήνει" σταδιακά
  • c = 20 (≈2√(km)) → Κρίσιμη — επιστροφή στο 0 όσο πιο γρήγορα γίνεται χωρίς ταλάντωση
  • c = 50 → Υπερκρίσιμη — πολύ αργή επιστροφή στο 0

Σημείο παρέμβασης εκπαιδευτικού · Η "ΑΗΑ!" στιγμή

Ρωτήστε τους μαθητές σας:

  1. «Γιατί η πόρτα με "αυτόματο κλείστρο" δεν χτυπάει δυνατά;» → κρίσιμη απόσβεση!
  2. «Γιατί το αμορτισέρ του αυτοκινήτου σας κάνει την ίδια δουλειά;» → ίδια αρχή!
  3. «Ποιο regime θα θέλατε σε σεισμόμετρο; Γιατί;» → υποκρίσιμη (για να καταγράφει το σήμα)
Πιθανή δυσκολία: Οι μαθητές μπορεί να μπερδέψουν "κρίσιμη" με "χειρότερη". Εξηγήστε ότι "κρίσιμη" = η βέλτιστη για γρήγορη ηρεμία.
Σύνδεση με BRaL Αυτό είναι model extension — η "καρδιά" του Refine. Οι μαθητές δεν διορθώνουν απλά bugs, αλλά εμπλουτίζουν το μοντέλο τους με νέες φυσικές αρχές. Έτσι μαθαίνουν ότι τα μοντέλα στη φυσική είναι πάντα απλοποιήσεις που βελτιώνονται.

07
◆ LEARN

Αναστοχασμός & σύνδεση με θεωρία

Reflection & theory connection
⏱ 10'

📝 Οδηγίες

Στο Claude (ίδια συνομιλία από πριν), επικολλήστε:

📋 PROMPT #5 · REFLECTION
Είμαστε τώρα στο LEARN phase του BRaL framework. Χρειάζομαι τη βοήθειά σου για αναστοχασμό. Βασισμένος στην προσομοίωση που φτιάξαμε (νόμος Hooke + damping), απάντησε: 1. ΤΙ ΑΛΛΑΞΕ: Σύγκρινε v1 (χωρίς damping) με v2 (με damping). Τι μαθησιακές ευκαιρίες προσφέρει κάθε έκδοση; 2. ΣΥΝΔΕΣΗ ΜΕ ΘΕΩΡΙΑ: - Ποια εξίσωση περιγράφει το σύστημα v2; - Ποια είναι τα 3 damping regimes και πώς σχετίζονται με την εξίσωση; - Γιατί η αρχική μετατόπιση ΔΕΝ επηρεάζει την περίοδο; (isochronism) 3. ΠΙΘΑΝΕΣ ΠΑΡΑΝΟΗΣΕΙΣ που η προσομοίωση μπορεί να αποκαλύψει: - Αναφέρε 3 κοινές παρανοήσεις μαθητών - Πώς η προσομοίωση τις διορθώνει; 4. ΜΕΤΑΦΟΡΑ (TRANSFER): - Πού αλλού στην καθημερινή ζωή συναντάμε την ίδια φυσική; - Δώσε 3 παραδείγματα με διαφορετικά damping regimes. Κράτα τις απαντήσεις σύντομες και κατάλληλες για μαθητές Α' Λυκείου.

✓ Τι αναμένουμε

Το Claude θα δώσει δομημένες απαντήσεις που μπορείτε να χρησιμοποιήσετε απευθείας στην τάξη:

  • 3 παρανοήσεις: βαρύτερη=γρηγορότερη, μεγαλύτερο πλάτος=μεγαλύτερη περίοδος, damping="θυσία" ακρίβειας
  • Παραδείγματα transfer: αμορτισέρ αυτοκινήτου (κρίσιμη), χορδή κιθάρας (υποκρίσιμη), πόρτα αυτόματου κλεισίματος (κρίσιμη)

Σημείο παρέμβασης εκπαιδευτικού

ΜΗΝ δεχτείτε τις απαντήσεις του AI παθητικά! Στην τάξη:

  1. Οι μαθητές γράφουν πρώτα τις δικές τους απαντήσεις (5 λεπτά)
  2. Μετά ζητάνε την απάντηση του AI
  3. Συγκρίνουν — ποιο έχει δίκιο; Υπάρχουν λάθη στο AI;
💡 Αυτό καλλιεργεί critical thinking απέναντι στο AI output — κρίσιμη δεξιότητα AI literacy (Wetzel & Kani, 2025).
Σύνδεση με BRaL Το Learn phase "κλείνει τον κύκλο" μέσω reflection και connection με θεωρία. Χωρίς αυτό, η δραστηριότητα παραμένει "τεχνικό παιχνίδι". «Δεν μετράει τι φτιάξαμε — μετράει τι μάθαμε φτιάχνοντάς το» (Sotiropoulos, 2026).
